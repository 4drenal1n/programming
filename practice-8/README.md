Контрольные вопросы
1. Зачем нужны include guards?

    Предотвращают повторное включение заголовочного файла при компиляции.

     1 #ifndef MODULE_H
     2 #define MODULE_H
     3 // код
     4 #endif

    Без них: ошибки повторного определения, конфликты имён.

    

2. Чем объявление отличается от определения?

    Объявление — сообщает компилятору об имени и типе, не выделяет память.

     1 extern int x;
     2 void func(int);

    Определение — создаёт объект/функцию, выделяет память.

     1 int x = 5;
     2 void func(int) { ... }

    Объявлений может быть много, определение — только одно.

    

3. Что такое статическая линковка?

    Код библиотеки встраивается в EXE-файл при компиляции.                                        
                                                                                                  
     1 # Создание библиотеки
     2 ar rcs libmath.a math.o
     3
     4 # Использование
     5 gcc main.c -L. -lmath -o program

    Плюсы: один файл, нет зависимостей при запуске
    Минусы: больший размер, нужно перекомпилировать при обновлении библиотеки

    

4. Как работает $^ в Makefile?

    $^ — все зависимости цели.

     1 program: main.o utils.o
     2     $(CC) $^ -o program
     3 # Развернётся в: gcc main.o utils.o -o program

    Другие переменные:                                                                            
     - $@ — имя цели                                                                              
     - $< — первая зависимость                                                                    
                                                                                                  
                                                                                               
5. Преимущества Meson перед Make?

     - Кроссплатформенный (работает на Windows, Linux, macOS)
     - Встроенная поддержка тестов (meson test)
     - Проще синтаксис (похож на Python)
     - Автоматическое отслеживание зависимостей
     - Генерация проектов для IDE (Visual Studio, Xcode)
     - Встроенная установка (meson install)
